# 数据集采集与实验 SOP

## 1. 数据集目标

建立一个小规模、可复现、面向毕业论文和 ICRA workshop 的 UAV 仿生变焦追踪数据集。

数据集定位：

- 不追求大规模。
- 强调真实平台、真实外场、真实 zoom 控制。
- 服务于 active optical zoom tracking 论文主线。

目标类别：

- 人：主目标。
- 标志桶：可控尺度目标。
- 足球框/球门：背景结构和场景锚点，可用于足球框背景下的人体追踪。

建议规模：

- 30-50 段视频。
- 总时长 1-2 小时。
- 每段 30-60 秒。

## 2. 数据存档原则

必须长期保存原始视频和元数据，因为毕业论文可能一年后再写。

每段实验建议目录：

```text
YYYYMMDD_site_height_target_motion_zoom_repeat/
├── raw_wide_or_4k.mp4
├── raw_zoom.mp4
├── metadata.jsonl
├── system_log.csv
├── experiment_config.yaml
└── notes.md
```

如果双路视频暂时无法稳定保存，至少保存：

- zoom 相机原始视频。
- zoom factor 时间序列。
- 实验配置。
- 系统日志。

## 3. metadata.jsonl 字段

每帧或关键帧记录一行 JSON：

```json
{"timestamp": 0.0, "frame_id": 1, "camera_source": "zoom", "zoom_factor": 5.0, "state": "Track", "bbox": [100, 120, 40, 90], "conf": 0.72, "target_class": "person", "tracking_status": "tracked", "latency_ms": 38.5, "fps": 24.7, "manual_or_auto_zoom": "auto"}
```

字段说明：

- timestamp：统一时间戳。
- frame_id：视频帧编号。
- camera_source：wide / zoom / 4k / default。
- zoom_factor：当前倍率。
- state：Search / Lock / Zoom / Track / Recover。
- bbox：目标框，格式为 x, y, w, h。
- conf：检测置信度。
- target_class：person / cone / goal。
- tracking_status：tracked / lost / recovered。
- latency_ms：端到端延迟。
- fps：实时 FPS。
- manual_or_auto_zoom：manual / auto / fixed。

## 4. system_log.csv 字段

建议字段：

```text
timestamp,cpu_percent,gpu_percent,memory_mb,decode_ms,inference_ms,tracking_ms,policy_ms,total_latency_ms,fps,dropped_frames
```

如果 Manifold 3 上暂时无法获取完整 GPU/NPU 占用，也至少记录：

- inference_ms。
- total_latency_ms。
- fps。
- dropped_frames。

## 6. 台架实验 SOP

目的：

在第一次外场前验证接口、存档、延迟和变焦响应，避免周末外场大量废片。

流程：

1. M400 + H30 + Manifold 3 上电，不起飞。
2. 启动 PSDK 程序。
3. 获取 Liveview 视频流。
4. 保存原始视频。
5. 执行 zoom 指令序列：

```text
1x -> 5x -> 10x -> 20x -> 10x -> 5x -> 1x
```

6. 记录每次指令发送时间和画面生效时间。
7. 记录视频帧率、丢帧、码率、延迟。
8. 检查 metadata 与视频是否能对齐回放。

台架实验输出：

- zoom response time 表。
- raw video + metadata 样例。
- 双流稳定性测试结论。
- 下一次外场风险清单。

## 7. 外场实验矩阵

高度：

- 20 m：系统验证和低风险样例。
- 40 m：主实验高度。
- 60 m：困难场景，只做少量补充。

倍率：

- Fixed 1x。
- Fixed 5x。
- Fixed 10x。
- Fixed 20x。
- Auto zoom。

目标与运动：

- 人横向行走。
- 人跑动。
- 人靠近/远离无人机。
- 人绕标志桶。
- 人从足球框附近经过。
- 静态标志桶多倍率采集。

## 8. 推荐外场顺序

### 第一轮外场：固定倍率采集

目标：

- 获取干净 baseline 数据。
- 不做复杂自动策略。

流程：

1. 20 m 标志桶静态，1x/5x/10x/20x。
2. 20 m 人横向走，1x/5x/10x。
3. 40 m 人横向走，1x/5x/10x。
4. 40 m 人靠近/远离，1x/10x。

### 第二轮外场：主动变焦试验

目标：

- 测试 Search-Lock-Zoom-Track-Recover 状态机。
- 测试自动 zoom 的响应和稳定性。

流程：

1. 20 m 人横向走，Auto。
2. 40 m 人横向走，Auto。
3. 40 m 人靠近/远离，Auto。
4. 40 m 人绕标志桶，Auto。

### 第三轮外场：完整对比实验

目标：

- 形成论文表格。

对比：

- Fixed 1x。
- Fixed 10x。
- Manual zoom。
- Auto zoom。

场景：

- 40 m 人横向走。
- 40 m 人靠近/远离。
- 40 m 人绕标志桶。
- 60 m 人横向走。

## 9. 标注策略

不要全量逐帧标注。

建议：

- 每段视频 5-10 fps 抽帧标注检测框。
- 关键跟踪片段连续标注。
- 优先标注 person。
- 标志桶可用于尺度标定。
- 足球框作为背景结构，可少量标注。

标注工具：

- CVAT。
- Label Studio。
- Roboflow。

输出格式：

- YOLO detection 格式。
- MOT tracking 格式。
- 自定义 JSONL 元数据。

## 10. 现场检查清单

起飞前：

- 电池充足。
- SD 卡/SSD 空间充足。
- Manifold 3 存储路径可写。
- PSDK 程序可启动。
- 视频能预览。
- metadata 正在写入。
- zoom 指令生效。
- 飞手和目标人员明确动作路线。

每段采集前：

- 口头报实验 ID。
- 确认高度。
- 确认目标类别。
- 确认运动方式。
- 确认 fixed / manual / auto zoom。

每段采集后：

- 检查视频文件大小非零。
- 检查 metadata 行数增长。
- 检查日志无明显中断。
- 记录异常。

停止条件：

- 风大导致画面严重抖动。
- 双流或主视频录制异常。
- metadata 中断。
- 目标出界超过 10 秒。
- 飞手认为不安全。

## 12. 最小成功标准

到 2026-09-15 前至少完成：

- 30-50 段可用视频。
- 一套可回放 metadata。
- 固定倍率 baseline。
- 自动变焦策略数据。
- Manifold 3 上 FPS 和延迟统计。
- 一段可展示的 demo 视频。
