# ICRA Workshop 小论文方案

## 1. 论文定位

目标会议：ICRA workshop。

论文风格：

- 强调真实机器人系统，而不是单纯视觉模型刷榜。
- 强调 active perception，即系统主动改变观测条件。
- 强调 Manifold 3 边缘部署与 H30 光学变焦闭环。
- 算法轻量、可解释、可复现，适合短周期完成。

建议标题：

> Uncertainty-Guided Foveated Zoom for UAV Small Object Tracking under Edge Computing Constraints

备选标题：

> UFZ-Track: Uncertainty-Guided Optical Zoom for UAV Small Object Tracking

## 2. 核心创新点

### Contribution 1：载荷级仿生变焦主动感知框架

提出一种面向 UAV 微小目标追踪的 foveated zoom tracking 框架。系统不控制 UAV 本体，而是通过 H30 光学变焦主动改变目标观测尺度。

### Contribution 2：不确定性驱动的变焦策略

基于检测置信度、目标像素面积、跟踪丢失帧数等指标构造不确定性分数，驱动 zoom in、hold、zoom out。

### Contribution 3：目标尺度保持控制律

设计轻量尺度保持策略，使目标维持在合适像素面积区间，避免固定广角目标过小，也避免长焦下目标出画。

### Contribution 4：真实边缘平台验证

在 M400 + H30 + Manifold 3 平台上完成视频采集、边缘推理、主动变焦和日志记录，验证系统可部署性。

## 3. 可参考的顶会思路

### FOVEA 类工作

思想：把有限视觉资源集中到感兴趣区域，提高小目标有效分辨率。

本课题转化：

- FOVEA 多为软件层面的图像放大或区域重采样。
- 本文使用 H30 光学变焦实现物理层面的 foveated observation。

### Fast-Tracker / Active Vision 类工作

思想：无人机追踪中，感知不是被动输入，而应主动调整观测条件。

本课题转化：

- 不控制 UAV 轨迹，降低外场风险。
- 只控制 H30 zoom，形成载荷级主动感知闭环。

### ByteTrack / OC-SORT 类工作

思想：

- ByteTrack：低置信度检测仍可能有助于目标恢复。
- OC-SORT：跟踪漂移时要依赖可靠观测修正。

本课题转化：

- Recover 阶段保留低置信度候选。
- 变焦前后使用最近可靠观测进行重关联。

## 4. Method 结构

方法名：UFZ-Track。

整体流程：

```text
Video frame
  -> Detector
  -> Tracker
  -> Uncertainty Estimator
  -> Foveated Zoom Policy
  -> H30 Zoom Command
  -> Re-association / Recover
```

### 4.1 Problem Formulation

给定 UAV 载荷视频流，系统在每一时刻观测：

```text
o_t = {I_t, b_t, c_t, z_t}
```

其中：

- I_t：当前图像帧。
- b_t：目标 bbox。
- c_t：检测置信度。
- z_t：当前 zoom factor。

系统动作：

```text
a_t ∈ {zoom_in, hold, zoom_out}
```

或连续形式：

```text
a_t = z_{t+1}
```

目标：

在边缘算力受限和视场受限条件下，最大化持续可观测性和跟踪稳定性。

### 4.2 State Machine

```text
Search -> Lock -> Zoom -> Track -> Recover
```

转移规则：

- Search 到 Lock：连续 N 帧检测到目标。
- Lock 到 Zoom：目标面积小于阈值或置信度不足。
- Zoom 到 Track：目标面积达到期望范围。
- Track 到 Recover：连续丢失 L 帧或置信度过低。
- Recover 到 Search：zoom out 后仍未找回目标。

### 4.3 Uncertainty Estimator

定义归一化目标面积：

```text
A_t = w_t h_t / (W H)
```

定义中心偏差：

```text
e_t = ||center(b_t) - center(I_t)|| / diag(I_t)
```

综合不确定性：

```text
U_t = λ1(1 - c_t) + λ2 max(0, A_min - A_t) + λ3 e_t + λ4 min(l_t / L, 1)
```

直觉：

- 目标越小，不确定性越高。
- 置信度越低，不确定性越高。
- 目标越靠近画面边缘，丢失风险越高。
- 连续丢失帧越多，需要 zoom out 恢复视场。

### 4.4 Scale-Preserving Zoom Controller

目标不是越放越大越好，而是维持目标像素面积在合适区间：

```text
A_min <= A_t <= A_max
```

目标倍率：

```text
z*_t = clamp(z_t * sqrt(A_ref / A_t), z_min, z_max)
```

平滑更新：

```text
z_{t+1} = z_t + clip(z*_t - z_t, -Δz, Δz)
```

实际策略：

- A_t < A_min 且目标稳定：zoom in。
- A_t 在区间内：hold。
- A_t > A_max 或目标接近丢失：zoom out。

### 4.5 Zoom-Aware Re-association

变焦过程会导致目标尺度和外观突变，普通 tracker 容易断链。

重关联分数：

```text
S = α IoU(b_pred, b_det)
  + β center_similarity
  + γ appearance_similarity
  + δ confidence
```

时间紧张时，appearance_similarity 可用 HSV histogram 或轻量特征替代深度 ReID。

### 4.6 Pseudocode

```text
for each frame I_t:
    detections = detector(I_t)
    track = tracker.update(detections)

    if no reliable target:
        state = Recover or Search
        z_cmd = zoom_out()
        continue

    A_t = bbox_area(track.bbox) / image_area
    U_t = uncertainty(track.conf, A_t, track.center_error, lost_count)

    if state in {Search, Lock} and target_confirmed:
        state = Zoom

    if U_t high and A_t < A_min:
        z_cmd = scale_controller(A_t, A_ref)
        state = Zoom
    elif target_lost:
        z_cmd = zoom_out()
        state = Recover
    else:
        z_cmd = hold()
        state = Track

    send_zoom_command(z_cmd)
    log(frame_id, state, bbox, conf, z_cmd, latency)
```

## 5. Baseline 设计

检测器候选：

- YOLOv8n / YOLOv8s。
- YOLO11n / YOLO11s。

跟踪器候选：

- ByteTrack：轻量，适合部署。
- BoT-SORT：更稳，但可能稍重。
- OC-SORT：对短时丢失和运动不规则更友好。

最终原则：

> 不是选 mAP 最高的模型，而是选 Manifold 3 上延迟、FPS、稳定性和精度最均衡的组合。
